# Jazzlore Metronome (`apps/metronome`) — v1 implementation plan

> Plan only. **No code touched.** On approval persisted to
> `apps/metronome/docs/plans/2026-05-20-v1-build.md`.

## Context

The populator's 47-hour enrichment run is in flight; while we wait we build
the next jazzlore sub-site: **`metronome.jazzlore.com`** — a web-based
metronome scoped to a very specific real-world problem (iPhone → USB-C → RCA
→ piano aux input) where native iOS metronomes drop the leading edge of
every click. A POC validated a Web-Audio approach that keeps a sub-audible
oscillator alive between clicks so the DAC never sleeps. This v1
productionizes that POC inside the Jazzlore monorepo using the same
conventions as `apps/scales`, `apps/chords`, `apps/musicians`.

Design fidelity is hi-fi: every size, weight, radius, letter-spacing, and
animation is committed in
`apps/metronome/docs/design_handoff_metronome/app/metronome-styles.css` and
`metronome-components.jsx`. The handoff README enumerates the visual rules
verbatim; the original spec enumerates the **non-negotiable audio
constraints** (keep-alive stream, 400ms warmup, Chris-Wilson lookahead
scheduler, dual Wake-Lock + NoSleep.js strategy, all kicked off
synchronously in the user gesture before any `await`).

Goal: ship v1 to prod on `metronome.jazzlore.com`, audible on iPhone-into-
Kawai-over-USB-C with no first-beat drop over a 5-minute session, ≤100 KB
gz initial bundle, axe-core 0 violations in both themes, Lighthouse perf
≥90 / a11y ≥95.

## Decisions locked in (before any code)

| # | Question | Decision | Why |
|---|---|---|---|
| 1 | Tone.js or raw Web Audio for the click engine? | **Raw Web Audio.** | The spec is very specific: 30 Hz keep-alive oscillator at ~0.0008 gain, Chris-Wilson 25 ms tick / 100 ms lookahead, two-voice wood-block (sine sweep + filtered noise). Tone.js Transport's abstractions get in the way and add ~75 KB gz; the metronome owns its own scheduler. `@jazzlore/music-core/audioEngine` stays untouched (still serves the sampler-based scales/chords playback). |
| 2 | Router? | **No router.** | "A single screen. No routes, no settings page." Drop `react-router` from deps — saves ~10 KB and removes a layer. `main.tsx` mounts `<Metronome />` directly. Cloudflare's `not_found_handling: "single-page-application"` still works (any URL serves `index.html`). |
| 3 | Theme toggle | **App-local icon button** that routes through `applyTheme` / `setOverride` / `resolveInitialTheme` from `@jazzlore/music-core`. | Per workspace memory, `@jazzlore/ui` `ThemeToggle` is frozen and not modified. Musicians already follows this pattern. The metronome's `.ic` 32 px header button matches the handoff CSS; identical hook wiring. |
| 4 | NoSleep.js — CDN or bundled? | **Bundled.** | Adds ~3 KB gz, eliminates CDN dependency, no CSP surface. Add `nosleep.js` to `apps/metronome/package.json` deps. |
| 5 | `primeAudio()` and metronome's own `AudioContext` | **Both.** Call `primeAudio()` from `@jazzlore/music-core` synchronously as the first statement of the Start handler; then construct the metronome's own `AudioContext` in the same gesture for the keep-alive oscillator + click voices. | `primeAudio()` flips `navigator.audioSession.type = 'playback'` (iOS ≥16.4) or runs the silent-`<audio>` fallback (pre-16.4) — that's what unlocks iOS USB audio routing. Once the session is `playback`, a fresh `AudioContext` in the same gesture starts in `running` state. Two contexts is fine: the music-core one is unused here. |
| 6 | Wake Lock strategy | **Dual — both engaged synchronously on Start, both released on Stop, `visibilitychange` re-acquires.** | Per spec. Wake Lock API alone is unreliable on Safari iOS; NoSleep's `video.play()` requires the user gesture to still be active, so neither can sit behind an `await`. |
| 7 | WKWebView banner | **In v1**, detect via user-agent and render a one-line banner above the BPM hero prompting "Open in Safari / Add to Home Screen". | Per spec + README — embedded webviews (Claude iOS, Slack, Discord, …) silently break both wake-lock layers. PWA manifest also ships in v1 so "Add to Home Screen" lands in a proper standalone context. |
| 8 | Pattern component shape | **`<PatternEditor rows={Row[]}>`** with v1 always rendering 1 row. | Forward-compatible per spec: v2 will add a second row for polyrhythms. Never bake in `rows.length === 1`. |
| 9 | `currentBeat` storage | **Ref + DOM `.flash` class, not React state.** | Per handoff: a 240 BPM run updates 240/min — re-rendering the whole tree on every beat is unnecessary churn. The scheduler calls `flashDot(i)` which adds `.flash` to the dot DOM node and removes it on `animationend`. Same for the BPM-hero pulse. |
| 10 | Persistence keys | `jazzlore:metronome:bpm`, `…:beats`, `…:pattern`, `…:mode` via `@jazzlore/music-core/storage`. | Mirror the existing storage discipline; never call `localStorage` directly. Status (idle/priming/running) is volatile, not persisted. Theme inherits the existing Jazzlore preference (`jazzlore:theme:v1`). |
| 11 | Visual baselines | Commit **light × dark × mobile × desktop** Playwright PNG baselines at the end of static-UI work. | Per `[[feedback-visual-baselines-before-refactor]]` — committed before any future refactor. For greenfield, ship them as the *initial* baseline so future passes have a reference. |
| 12 | Deploy shape | New `wrangler.metronome.jsonc` at repo root, assets-only (clone `wrangler.chords.jsonc`). `name: "jazzlore-metronome"`, `directory: "./apps/metronome/dist"`. | No backend; SPA only. Custom domain binding (`metronome.jazzlore.com`) is a one-time Cloudflare dashboard step — first deploy lands at `jazzlore-metronome.workers.dev` and is verified there before DNS is pointed. |

## State model

```ts
type BeatState = 'empty' | 'normal' | 'accent'
type Mode = 'all' | 'backbeat' | 'altmeasure' | 'custom'
type Status = 'idle' | 'priming' | 'running'

interface MetronomeState {
  bpm: number              // 30..240
  beats: number            // 2..7
  pattern: BeatState[]     // pattern.length === beats invariant
  mode: Mode
  bpmEditing: boolean
  tapArmed: boolean
  status: Status
  altMeasureSilent: boolean  // runtime flag for mode === 'altmeasure'
}

type Action =
  | { type: 'SET_BPM'; bpm: number }
  | { type: 'NUDGE_BPM'; delta: number }
  | { type: 'CLASSIC_STEP'; dir: 'prev' | 'next' }
  | { type: 'SET_BEATS'; beats: number }
  | { type: 'CYCLE_DOT'; index: number }
  | { type: 'SET_MODE'; mode: Mode }
  | { type: 'BPM_EDIT_START' } | { type: 'BPM_EDIT_COMMIT' } | { type: 'BPM_EDIT_CANCEL' }
  | { type: 'TAP'; t: number }
  | { type: 'TAP_DISARM' }
  | { type: 'START' } | { type: 'PRIMED' } | { type: 'STOP' }
```

**Reducer rules** (each gets a unit test):

- `CYCLE_DOT` → `empty → normal → accent → empty`; **clears active mode** (sets `mode: 'custom'`) when a quick mode was on.
- `SET_BEATS` → pad with `'normal'` or truncate the pattern to keep length === beats. **If a quick mode is on, reapply it** rather than untoggle.
- `SET_MODE` → applies the mode's pattern (all → `['accent', ...'normal' × (beats-1)]`; backbeat → odd-indexed `'normal'`, even `'empty'`; altmeasure → pattern unchanged + sets `altMeasureSilent: false` initially).
- `NUDGE_BPM` → clamps to [30, 240].
- `CLASSIC_STEP` → jumps to next-lower/-higher classic tempo in `[40,60,66,76,108,120,144,168,200]`; clamps at boundaries.
- `TAP` → tap-averager accumulates timestamps; reset if >2 s gap; once ≥2 taps, commit BPM = round(60_000 / mean-gap-of-last-6).

**Persistence** — debounced (~200 ms) writes of `{bpm, beats, pattern, mode}` to `jazzlore:metronome:*`. Rehydrate on mount via `read()`; if any field is malformed, fall back to the v1 defaults (`bpm: 120, beats: 4, pattern: ['accent','normal','normal','normal'], mode: 'all'`).

## Audio engine

```
features/audio/engine.ts
├── createMetronomeEngine(): {
│     start(): Promise<void>       // primeAudio() first, then construct ctx + warmup + scheduler
│     stop(): void                 // stop scheduler, ramp keep-alive to 0 over 30ms, close ctx after 200ms
│     setBpm(bpm): void
│     setPattern(p, mode, altMeasureSilent): void
│     onBeat(cb): void             // fires (beatIndex, audioTime) — UI uses for flash
│   }
└── all internal:
    ├── keepAlive (30 Hz osc → gain 0.0008 → ctx.destination)
    ├── scheduler.ts (lookaheadMs=100, intervalMs=25; Chris-Wilson)
    ├── voices.playWoodBlock(ctx, time, accented):
    │     - accented: 1.1 kHz sine sweep down to 800 Hz over 25 ms + filtered noise burst
    │     - normal: 880 Hz sine sweep down to 660 Hz over 18 ms + filtered noise burst
    │     - master gain ~0.4 (fixed; no volume control in v1)
    └── 400 ms warmup window: scheduler starts; first beat scheduled at ctx.currentTime + 0.4
```

`engine.start()` is **`async`** but the *first synchronous step inside the
caller* (StartStopButton's onClick) is `primeAudio()` from
`@jazzlore/music-core` — before any `await engine.start()`. Same gesture
also fires `wakeLock.request()` + `noSleep.enable()`. Pattern (copy
verbatim from the spec):

```tsx
function onStartClick() {
  primeAudio()                                 // 1. sync — unlocks iOS audio session
  wakeLock.requestSync()                       // 2. sync — kicks both Wake Lock + NoSleep before any await
  void engine.start().then(() => dispatch({type:'PRIMED'}))
}
```

**Mode gating in the scheduler:**
- `mode === 'altmeasure'`: maintain a `measureCount` ref; on every bar boundary toggle `currentMeasureSilent`. When silent, schedule **no** clicks for that bar AND don't fire `onBeat` (so the visible pattern row doesn't flash — but the second dimmed "bar 2 — silent" preview row stays rendered in DOM).
- `pattern[i] === 'empty'`: skip the click, but **still fire** `onBeat(i)` so the dim/dashed dot can flash if desired (per spec the empty dots stay visible during running; their flash is subtle).
- `pattern[i] === 'normal'` vs `'accent'`: choose voice.

## Keyboard shortcuts

A single `useKeyboardShortcuts(dispatch, engineRef)` hook listens on
`window` for `keydown`:

| Key | Action |
|---|---|
| `space` | toggle start/stop — call the same handler the Start button uses, including `primeAudio()` |
| `T` | dispatch `TAP` (records `performance.now()`) |
| `←` / `→` | dispatch `NUDGE_BPM` ±1; repeat-on-hold |
| `Shift + ←` / `Shift + →` | dispatch `NUDGE_BPM` ±10; repeat-on-hold |
| `[` / `]` | dispatch `CLASSIC_STEP` prev/next |
| `1`…`7` | dispatch `SET_BEATS` (only `2`–`7` valid; `1` ignored since current spec says 2..7) |
| digit keys when `bpmEditing` | accumulate; `Enter` commits, `Esc` cancels |

Shortcuts no-op when focus is in `<input>` / `<textarea>` to avoid trapping
typing.
